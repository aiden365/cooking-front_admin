# 智能菜谱系统 (Intelligent Recipe System) - 管理后台开发指南

## 1. 项目概述

本项目是“智能菜谱系统”的管理后台端，主要供系统管理员使用。
项目基于 `vue-pure-admin` (精简版/thin版本) 开发，核心功能涵盖：

菜谱管理
  菜谱列表 包含添加，编辑，删除，标签维护功能
	菜谱评价列表	包含编辑功能
用户管理
	用户列表 包含添加，编辑，删除，标签维护，锁定/解锁功能
	用户分享 包含查看、删除功能
	饮食记录 日期，餐次，菜品名
	营养目标 包含添加，编辑，删除功能
知识库管理
	知识列表 包含添加，删除功能
标签管理
	菜谱标签 包含添加，编辑，删除功能
	用户标签 包含添加，编辑，删除功能
系统管理
	管理员管理 包含添加，编辑，删除，启用/禁用功能
	参数配置 （页面以卡片列表形式展示）营养数量参数配置，大模型KEY参数配置


## 2. 技术栈约束与规范

在生成、修改代码时，请严格遵守以下技术栈和框架规范：

- **核心框架:** Vue 3 (强制使用 Composition API 和 `<script setup>` 语法)
- **开发语言:** TypeScript (必须定义完整的 interface/type，禁止随意使用 `any`)
- **UI 组件库:** Element Plus (优先使用其提供的 Table, Form, Dialog, Pagination 等组件)
- **CSS 方案:** 使用 Tailwind CSS 用于布局、间距和基础排版，也可以手写 CSS 类也可以使用
- **路由与权限:** 依赖 `vue-pure-admin` 内置的动态路由和权限管理机制
- **网络请求:** Axios (已在框架底层封装，统一放置在 `src/api` 目录下)
- **图标库:** 优先使用框架内置的 Iconify 图标方案或 Element Plus 官方图标

## 4. Mock 数据编写规范
由于当前不连接真实后端，生成涉及数据的业务功能时，必须包含相应的 Mock 文件代码。
1. **Mock 结构:** 必须在 `src/mock/` 目录下创建对应的 `.ts` 文件，使用 `export default []` 导出由 `MockMethod` 对象组成的数组。
2. **响应格式:** 模拟接口的返回值必须符合统一的标准格式。例如：
   `{ success: true, code: 200, data: { ... }, message: "请求成功" }`
3. **接口定义:** 在 `src/api/` 中正常编写 Axios 请求方法，URL 路径需与 `src/mock/` 中拦截的 URL 一致，确保未来对接真实后端时，只需删除 Mock 文件即可，无需大量修改组件代码。
